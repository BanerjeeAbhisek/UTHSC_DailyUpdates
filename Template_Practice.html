Why we need richer variational posteriors

Variational inference (ELBO)

We approximate the intractable posterior with a tractable family:

q_\phi(z\mid x) \approx p_\theta(z\mid x)

[ p_(x)   {q(zx)}[p_(xz)] -!(q_(zx),|,p(z)) ]

In practice: choose a simple (q_) so gradients are cheap.

The bottleneck: limited (q_)

Common choice (fast, but restrictive):

  • Mean-field / diagonal Gaussian
    (q_(zx)=((x),(^2(x))))

Issues reported in VI: - misses dependencies / multimodality - often underestimates posterior variance - can bias parameter estimates

So even with infinite data/compute, VI may not match (p_(zx)).

Geometric intuition (visual)

chart = {
  // d3 is available globally from include-in-header
  const W = 800, H = 420, pad = 30;

  const x = d3.scaleLinear().domain([-2,2]).range([pad, W-pad]);
  const y = d3.scaleLinear().domain([-1.4,1.4]).range([H-pad, pad]);

  const pts = [];
  for (let xx=-2; xx<=2.001; xx+=0.18) {
    for (let yy=-1.4; yy<=1.4001; yy+=0.18) {
      pts.push([xx,yy]);
    }
  }

  const svg = d3.create("svg")
    .attr("viewBox", `0 0 ${W} ${H}`)
    .style("width","100%")
    .style("height","420px")
    .style("border","1px solid rgba(0,0,0,0.15)")
    .style("border-radius","12px");

  svg.selectAll("circle")
    .data(pts)
    .join("circle")
      .attr("r", 2)
      .attr("opacity", 0.7)
      .attr("fill", "#E78824")
      .attr("cx", d => x(d[0]))
      .attr("cy", d => y(d[1]));

  return svg.node();
}
